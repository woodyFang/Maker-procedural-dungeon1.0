# AI 题材资产生成与区域细节设计

## 1. 文档目的

本文冻结两条核心产品决策：

1. AI 不以现有模型、物件和材质 ID 为输入上限；AI 根据题材生成所需规则，并生成该题材需要的 3D 模型、材质和最终摆放结果。
2. “12 张课桌、整齐排列、黑板靠墙”等精确数量与排列要求属于具体区域的高优先级细节调整，不写死在题材组或房间组中。

本文描述目标方案。当前实现仍以本地规则规划器和已有题材包物件为主，尚未完成 3D 模型生成、导入和最终实例保存管线。

## 2. AI 的输入与最终产物

### 2.1 AI 输入

AI 接收：

- 题材名称和题材提示词。
- 可选参考图片，用于辅助判断造型、材质、色彩和氛围。
- 可选房间组定义，例如教室、图书馆、实验室。
- 当前场景的楼层、区域轮廓、墙面、门口、通道和可用高度。
- 质量、数量、性能和模型复杂度预算。
- 用户已锁定的模型和区域设置。

现有素材 ID 不作为题材创作的硬限制。AI 生成模型成功后，系统再为新模型分配稳定的 `assetId`。

### 2.2 AI 最终产物

一次完整题材生成应产生一个可保存、可编辑、可复现的题材生成包：

```text
ThemeGenerationPackage
├─ 题材全局风格与生成规则
├─ 题材通用素材定义
├─ 可选房间专属素材定义
├─ AI 生成的 3D 模型文件
├─ AI 生成或配置的材质文件
├─ 模型尺寸、中心点、语义标签和碰撞信息
├─ 通用摆放规则
├─ 房间专属摆放规则
├─ 区域细节覆盖规则
├─ 已计算完成的场景实例
└─ 生成种子、题材版本和接口版本
```

最终保存的不只是自然语言或摆放建议，而是已经具备 `assetId`、坐标、旋转和缩放的场景实例。再次加载时直接恢复结果，不重复调用 AI。

## 3. AI 的完整工作链路

```text
题材提示词/参考图
→ 理解题材语义
→ 生成题材规则
→ 规划通用与房间专属素材
→ 调用 3D 模型生成接口
→ 下载并转换模型与材质
→ 统一米制尺寸、中心点和碰撞信息
→ 分配稳定 assetId
→ 生成摆放规则
→ 结合真实区域几何计算最终位置
→ 保存题材包、模型文件和场景实例
```

题材规划 AI 负责决定需要什么；3D 生成接口负责产生模型文件；摆放执行器在 AI 规则约束下根据真实几何完成坐标、碰撞和门口避让。三者共同构成对用户可见的“AI 生成并摆放”流程。

### 3.1 AI 介入的三类职责

AI 不是一次调用完成全部工作，而是分成三个职责明确的阶段：

| AI 职责 | 主要输入 | 主要输出 |
|---|---|---|
| 题材规划 AI | 题材名称、提示词、参考图、可选房间定义、性能预算 | 题材规则、通用素材需求、房间专属素材需求、生成任务清单 |
| 3D 资产生成 AI | 单个素材的名称、用途、造型要求、参考图、尺寸和质量约束 | 3D 模型、材质贴图和生成元数据 |
| 区域布局 AI | 已生成素材、区域几何、房间组、区域细节要求、锁定实例 | 通用/房间素材选择、逻辑布局意图和待落位实例清单 |

布局 AI 负责“放什么、放几个、形成什么空间关系”；确定性摆放执行器负责把意图换算成合法坐标，并完成碰撞、墙面吸附、门口避让和最终实例创建。最终用户看到的是 AI 已经生成模型并完成摆放的场景。

### 3.2 从用户输入到已摆放场景的完整流程

#### 阶段 A：创建生成任务

触发方式：

- 用户新建或重新生成题材。
- 用户修改题材提示词或参考图。
- 用户新增、删除或修改房间组。
- 用户选中具体区域并要求 AI 生成或重新布局。

系统先生成稳定的 `jobId` 和输入版本，保存当前题材、房间、区域和锁定实例快照。之后收到的 AI 结果必须与该版本一致，过期结果不得覆盖新编辑。

#### 阶段 B：题材规划 AI

AI 获得：

- 题材名称、提示词和可选参考图。
- 用户是否定义了房间组，以及每个房间组的自然语言说明。
- 目标平台的模型数量、面数、材质尺寸和性能预算。
- 当前区域数量、面积范围和楼层等场景摘要。

AI 生成：

- 题材整体风格、材质、色彩和灯光方向。
- 无房间也能使用的题材通用素材需求。
- 已定义房间对应的专属素材需求。
- 每种素材的用途、建议尺寸、变体数量、生成提示词和质量要求。
- 通用、房间和区域三层规则的初始版本。

这一阶段只生成题材方案和资产任务，不输出虚假的模型路径，也不直接写入场景。

#### 阶段 C：建立 3D 资产生成队列

系统把规划结果拆成独立资产任务：

```text
通用资产任务
├─ 公告栏
├─ 门牌
├─ 饮水机
└─ 顶灯

房间资产任务（仅为已定义房间创建）
├─ 教室：课桌、黑板、讲台
└─ 实验室：实验台、仪器柜
```

系统先检查相同题材版本和资产需求是否已有可复用缓存；只有缺失或已失效的资产进入 AI 生成队列。

#### 阶段 D：3D 资产生成 AI

每个资产任务分别调用 3D 生成接口。AI 获得：

- 资产名称和场景用途。
- 题材风格与参考图片。
- 米制目标尺寸和比例。
- 需要的正面、背面、落地面和墙面安装语义。
- 面数、贴图尺寸、材质数量和变体数量预算。

AI 输出模型文件、材质贴图和生成元数据。生成失败的任务单独重试，不能让一个模型失败导致整个题材数据丢失。

#### 阶段 E：确定性资产导入与校验

此阶段不由语言模型猜测，系统负责：

- 将模型转换成引擎支持的格式。
- 统一为米制单位和正确坐标系。
- 修正模型中心点、落地点和朝向。
- 检查模型边界、材质和贴图完整性。
- 生成或校验碰撞信息。
- 按性能预算检查面数、材质数和贴图尺寸。
- 为通过校验的模型分配稳定 `assetId`。

未通过校验的模型进入修复或重新生成任务；只有合法资产可以进入布局阶段。

#### 阶段 F：区域布局 AI

布局必须在区域几何和资产可用后进行。AI 获得：

- 当前区域的轮廓、面积、墙面、门口、通道和高度。
- 当前区域绑定的房间组；没有绑定时为空。
- 题材通用素材池。
- 当前房间的专属素材池。
- 区域细节覆盖要求。
- 已锁定、禁止移动或必须保留的实例。
- 随机种子。

布局 AI 按以下顺序工作：

```text
读取题材通用规则
→ 若有房间组，叠加房间专属规则
→ 若有区域细节，按最高优先级覆盖
→ 决定本区域使用哪些模型及空间关系
→ 输出待落位实例和逻辑锚点
```

没有房间定义时，只使用题材通用素材和通用布局逻辑；有房间定义时，随机组合题材通用素材与房间专属素材。区域细节中的精确数量、行列和墙面要求覆盖前两层规则。

#### 阶段 G：确定性摆放执行

摆放执行器将 AI 的逻辑布局转换为最终实例：

- 计算精确坐标、旋转和缩放。
- 完成地面、墙面和中心线吸附。
- 避开门口、通道、楼梯和其他模型。
- 检查模型之间和模型与房间边界的碰撞。
- 在空间不足时返回明确冲突，不静默重叠或删减用户的精确要求。
- 为每个实例记录来源：通用规则、房间规则或区域覆盖。

#### 阶段 H：预览、确认与局部重算

系统展示已经导入并摆放的真实模型预览，而不是占位框。用户可以：

- 接受整个题材结果。
- 只重新生成某个模型。
- 只重新布局当前区域。
- 锁定满意的模型或区域。
- 修改区域细节后重新计算当前区域。

局部修改不得触发整个题材无差别重做。

#### 阶段 I：提交与保存

用户确认后，系统一次性保存：

- 原始题材输入和参考图信息。
- 题材、房间与区域三层规则。
- 3D 模型、材质及资产清单。
- 每个区域的最终实例和变换数据。
- 锁定状态、生成种子和各层版本号。
- AI 接口、模型和提示词版本，用于问题追踪与回测。

保存成功后状态才进入 `ready`，场景生成与重新加载直接使用已保存实例。

### 3.3 任务状态

完整 AI 任务建议使用以下状态：

```text
draft
→ planning
→ generating_assets
→ importing_assets
→ planning_layout
→ placing
→ preview
→ ready
```

任一阶段失败进入 `failed`，同时保存失败阶段、资产任务和错误原因。已经生成并通过校验的模型应保留，重试时从失败节点继续，不从头重复计费生成。

### 3.4 后续编辑时 AI 的介入范围

| 用户操作 | AI 重新介入范围 |
|---|---|
| 修改题材提示词或参考图 | 重新规划题材；仅重生成受影响且未锁定的资产与区域 |
| 新增房间组 | 生成该房间的专属资产与规则；不重做无关房间 |
| 修改房间组 | 更新使用该房间组且未锁定的区域 |
| 给区域分配房间组 | 为该区域执行房间布局 AI 和确定性摆放 |
| 修改“12 张课桌、3×4 排列”等区域细节 | 只重新计算当前区域 |
| 锁定模型 | 后续 AI 不得删除、替换或移动该实例 |
| 删除房间组 | 清理该组专属规则；通用题材素材仍然有效 |

## 4. 没有房间定义时

题材组必须能够独立运行。只有题材提示词、没有房间组时，AI 生成题材通用素材和通用规则，并在所有合适区域随机摆放。

以“学校”为例，通用素材可以包括：

- 门牌和校园导视。
- 公告栏。
- 长椅和饮水机。
- 垃圾桶和消防设备。
- 学校风格的灯具、墙面装饰和公共设施。

这条路径不强制把区域解释为教室、图书馆等具体房间。

```text
题材通用素材池
→ 按区域面积、墙面、门口和通道筛选候选位置
→ 按权重、密度和随机种子选择素材
→ 完成碰撞检查与摆放
```

## 5. 有房间定义时

存在房间组时，每个具体区域执行两层素材生成与摆放：

1. 先应用题材通用素材池。
2. 再应用该房间组的专属素材池。

例如“学校 / 教室”区域：

```text
学校通用池：门牌、公告栏、垃圾桶、顶灯
+
教室专属池：课桌、椅子、讲台、黑板、储物柜
→ 按权重、空间容量和随机种子组合
→ 生成最终实例
```

房间组只描述语义和宽松默认值，例如“教室通常包含课桌、讲台和黑板”“课桌通常面向授课墙面”。它不默认写死某个具体区域必须有 12 张课桌。

## 6. 三层规则模型

### 6.1 题材组：全局层

题材组负责：

- 整体风格、色彩、材质和灯光倾向。
- 题材通用模型的生成需求。
- 通用素材的密度、权重和摆放倾向。
- 所有区域共享的基础规则。

题材组不负责某个具体区域的精确物件数量和行列布局。

### 6.1.1 题材模型生成规则

题材模型生成必须先输出可执行的资产规格，再调用 3D 模型生成接口。每个模型任务至少包含：

- 语义用途和所属层级：题材通用、房间专属或区域覆盖。
- 真实尺寸、落地点、中心点和前方向；长度单位统一为米。
- 造型、材质、颜色和灯光要求，以及允许的变体数量。
- 边界盒、碰撞形状、材质槽、面数、贴图尺寸和 LOD 性能预算。
- 稳定的 `assetId`、生成种子、来源提示词和版本信息。

模型生成遵循以下约束：

1. 不生成占位框、虚假的资源路径或无法导入的模型；生成失败只影响对应资产任务。
2. 题材层只决定通用模型需求和风格，房间层决定专属模型需求，区域层才允许指定精确数量、行列和朝向。
3. 导入时必须按真实边界盒修正中心点、落地点和朝向，不能用模型名称猜测尺寸。
4. 模型、材质、碰撞和性能元数据必须一起保存；未完成导入校验的模型不能进入布局阶段。

### 6.1.2 题材颜色规则

颜色采用语义色板，不允许模型生成接口为每个物件任意发明孤立颜色。颜色字段统一使用 `#RRGGBB`，至少包含：

`floor`、`corridor`、`wall`、`pillar`、`accentObject`、`cloth`、`flame`、`flameCore`。

规则如下：

- `floor`、`corridor`、`wall` 和 `pillar` 负责空间层级区分；相邻结构必须保持可读的明度或色相差异。
- `accentObject` 用于交互物、重点道具和题材识别，不得覆盖整个场景的结构色。
- `cloth` 用于布料、软装和装饰；`flame`、`flameCore` 只用于发光体及其核心高光。
- 墙帽、碎片、火焰灯光和粒子颜色由语义色板派生，不能绕过色板单独写死。
- 颜色只重配模型材质；题材环境的雾色、天空、地面和灯光仍由题材/调色板共同决定，不能把“颜色”误当成几何规则。

颜色验收必须以最终渲染画面为准，而不是只检查输入色值。对排除 UI 后的场景像素计算归一化 sRGB 亮度：

`Y = 0.2126R + 0.7152G + 0.0722B`

默认画面亮度边界为：平均亮度 `0.12` 至 `0.82`、压黑像素比例不超过 `8%`、过曝像素比例不超过 `3%`。有意使用的 `flame`、`flameCore` 和其他发光材质可以超过普通亮度上限，但必须单独标记并限制覆盖面积，不能用发光区域掩盖整体过黑或让画面大面积纯白。

### 6.1.3 统一验收检查规则

新增一条统一验收门槛：**只有模型已导入、材质和贴图完整、尺寸/中心点/碰撞有效、颜色字段和最终渲染亮度通过色板校验，并且摆放通过房间边界、门口、通道和模型间碰撞检查时，题材或资产才能进入 `ready`。**

验收失败必须记录阶段、资产任务、检查项和可读原因；不得静默重叠、堵塞通道、用占位模型继续生成，也不得把部分通过的结果标记为 `ready`。已通过的独立资产可以保留，重试从失败检查项继续。

### 6.2 房间组：类型层

房间组负责：

- 定义房间用途和语义。
- 生成该类型房间需要的专属 3D 模型。
- 定义可用素材池和宽松布局倾向。
- 提供密度、权重或自动数量范围。

例如教室房间组可以说明课桌通常成组排列、黑板适合靠墙，但不固定课桌数量。

### 6.3 区域设置：实例层

选中二维或三维场景中的具体区域后，用户可以进行最高优先级的细节调整：

- 精确物件数量。
- 行数、列数和排列方式。
- 行列间距。
- 面向方向和目标墙面。
- 靠墙、居中、沿边或成组等锚点。
- 必须保留、禁止出现或锁定的模型。
- 该区域独立的自然语言补充要求。

例如：

```text
区域：第 1 层 / 区域 12
房间组：教室
课桌：12 张，3 行 × 4 列，整齐排列
方向：全部面向前墙
黑板：前墙居中
讲台：黑板前方居中
```

规则优先级固定为：

```text
区域细节覆盖
> 房间组规则
> 题材组规则
> AI 自动默认值
```

## 7. 区域自动布局与详细布局

### 7.1 区域没有详细设置

AI 和摆放执行器根据区域几何自动决定：

- 合理的模型数量。
- 行列与间距。
- 适合放黑板、柜体或装饰的墙面。
- 门口、通道和活动空间的避让。

### 7.2 区域存在详细设置

系统优先满足用户要求，并先进行容量和碰撞检查：

```text
详细要求
→ 检查区域是否容纳
→ 计算候选布局
→ 检查门口、通道和模型碰撞
→ 生成预览
→ 用户确认后保存最终实例
```

如果区域无法容纳指定数量，系统必须说明冲突原因并提供可选调整，不能静默重叠模型或堵塞门口。

## 8. 建议数据边界

房间组保存宽松规则，精确要求保存到独立的区域覆盖数据中：

```json
{
  "regionId": "floor1-region12",
  "roomGroupId": "classroom",
```

## 9. 保存与再生成原则

- 保存题材规则、房间规则、区域覆盖规则、生成模型文件和最终实例。
- 保存题材版本、区域版本和随机种子，保证结果可回测。
- 修改题材时只重新生成受影响且未锁定的素材与区域。
- 修改房间组时只影响使用该房间组且未锁定的区域。
- 修改区域细节时只重新布局当前区域。
- 用户锁定的模型实例和区域结果不得被后续 AI 自动覆盖。

## 10. 当前实现需要调整的方向

当前 `roomGroups.propRules.count` 不应继续承担所有精确数量控制。目标调整为：

- 题材组保存通用素材生成需求和通用规则。
- 房间组保存专属素材生成需求、权重、密度和宽松范围。
- 新增 `regionOverrides` 保存精确数量与排列要求。
- 新增 AI 3D 素材生成、下载、转换、校验和稳定 ID 分配管线。
- 新增最终场景实例保存，确保模型生成后已经完成摆放。
- 保留随机种子，使通用和房间素材的组合可复现、可回测。

最终用户流程为：

```text
输入题材和参考图
→ AI 生成题材规则、3D 模型和材质
→ 无房间定义时使用题材通用素材完成摆放
→ 有房间定义时组合通用素材与房间专属素材
→ 用户选择具体区域进行精确细节调整
→ 重新生成当前区域并确认
→ 保存完整题材包和已摆放场景
```
